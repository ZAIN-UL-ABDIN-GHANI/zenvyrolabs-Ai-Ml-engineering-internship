FRONTEND OVERVIEW

HTML pages
- frontend/index.html is the backend-connected production-style UI.
- frontend/demo.html is the standalone mock simulation UI.

JavaScript files
- frontend/js/app.js drives the production flow and makes API calls to the backend.
- frontend/js/demo.js drives the local mock debate simulation without backend calls.

CSS files
- frontend/css/demo.css contains the primary visual styling used by both HTML pages.
- frontend/css/styles.css is an additional CSS file with an alternate style system and older class names.

User interface components
- Top bar with logo, title, and action controls.
- Topic banner showing the current debate topic.
- Two agent panels for the advocate and challenger.
- Feed areas for rendered messages.
- Status bar for current activity and connection state.
- Judge overlay / modal for the final verdict display.
- Timer, round counter, and score cards in the demo page.

API calls
- frontend/js/app.js calls POST http://127.0.0.1:5000/api/debate/start.
- frontend/js/app.js calls POST http://127.0.0.1:5000/api/debate/next-turn.
- frontend/js/app.js does not call the machine-learning routes directly in the current code.
- frontend/js/demo.js does not make backend API calls.

Overall frontend workflow
- index.html loads app.js and shows a topic input for backend-connected debate orchestration.
- The user enters a topic and starts the debate from the UI.
- app.js sends the topic to the backend, receives a JSON response, and renders the first message.
- Subsequent turns are requested through the next-turn endpoint and appended to the agent feeds.
- demo.html loads demo.js and runs a timed mock debate with preset argument text.
- demo.js updates the clock, alternates between the two agents, and shows a simulated judge verdict at the end.
